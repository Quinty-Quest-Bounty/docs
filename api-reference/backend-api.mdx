## Base URL

```
https://be-quinty.fly.dev    # Production
http://localhost:3001     # Local development
```

## Authentication

The API uses JWT tokens stored in httpOnly cookies. After calling `/auth/sync-profile` or `/auth/exchange`, a cookie named `auth-token` is set automatically.

Authenticated endpoints require this cookie. The cookie is valid for 7 days.

## Endpoints

### Auth

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| `POST` | `/auth/sync-profile` | No | Sync user profile from Privy and get JWT |
| `POST` | `/auth/exchange` | No | Exchange Supabase token for app JWT |
| `GET` | `/auth/me` | Yes | Get current authenticated user profile |
| `PATCH` | `/auth/profile` | Yes | Update user profile fields |
| `POST` | `/auth/link-wallet` | Yes | Link a wallet address to the account |
| `POST` | `/auth/logout` | No | Clear the auth cookie |
| `POST` | `/auth/twitter/callback` | No | Exchange Twitter OAuth code for user data |

### Users

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| `GET` | `/users/wallet/:address` | No | Look up a user by wallet address |
| `GET` | `/users/:id` | Yes | Get user profile by Privy ID |

## Key Endpoints

### POST /auth/sync-profile

The primary authentication endpoint. Called by the frontend after Privy login.

**Request body:**

```json
{
  "id": "did:privy:xxx",
  "email": "user@example.com",
  "walletAddress": "0x...",
  "name": "User Name"
}
```

**Response:**

```json
{
  "success": true,
  "profile": { ... },
  "access_token": "eyJ..."
}
```

Also sets the `auth-token` httpOnly cookie.

### POST /auth/twitter/callback

Exchanges a Twitter OAuth authorization code for user data using the PKCE flow.

**Request body:**

```json
{
  "code": "oauth_authorization_code",
  "code_verifier": "pkce_code_verifier",
  "redirect_uri": "https://app.quinty.io/auth/callback"
}
```

**Response:**

```json
{
  "userId": "123456789",
  "username": "johndoe"
}
```

### POST /auth/link-wallet

Links a wallet address to the authenticated user's account. Requires a signed message for verification.

**Request body:**

```json
{
  "walletAddress": "0x...",
  "signature": "0x..."
}
```
